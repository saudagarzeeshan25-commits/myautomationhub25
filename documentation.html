Free Consultation

Every engagement starts with a free 30-minute strategy call. This call is for me to understand your business and for you to understand whether automation is the right solution.

What to prepare

  • A list of tasks or processes you currently do manually that you'd like to automate
  • The tools and platforms your business currently uses (CRM, email, spreadsheets, etc.)
  • A rough idea of how often the task happens (daily, weekly, per new customer, etc.)
  • Any specific outcome you're hoping to achieve (save time, reduce errors, increase speed, etc.)

Tip: You don't need to know exactly what you want automated. Just describe your problem and I'll help identify the best automation approach.

Scoping a Project

After the consultation call, I provide a written project scope document. This is the single most important document in the project ‚Äî it defines exactly what will be built.

What the scope includes

  • A plain-English description of what the automation does
  • The trigger (what starts the automation)
  • The actions (what happens as a result)
  • Tools and platforms involved
  • Any conditions or logic applied
  • Number of included revisions
  • Project price and payment terms
  • Estimated delivery date

‚ö†Ô∏è Anything not included in the scope will be treated as a new request and quoted separately. Review the scope carefully before confirming.

Delivery & Testing

When your automation is ready, I'll arrange a walkthrough session where I demonstrate exactly how it works. I'll explain each step and answer any questions you have.

What's delivered

  • The fully working automation (configured in your accounts)
  • A plain-language explanation of how it works
  • Instructions for basic troubleshooting
  • Guidance on any manual steps still required
  • Handover of any credentials or access used during the build

Your testing responsibility

After delivery, I ask that you test the automation with real data within 3‚Äì5 business days and report any issues. Bugs found within this window are fixed at no charge.

Post-Delivery Support

Every project includes a 7-day post-delivery support window. During this time, any bugs or issues within the agreed scope are resolved at no extra charge.

What's covered

  • Bugs or errors that prevent the automation from functioning as scoped
  • Minor adjustments to logic or flow within the original scope
  • Guidance on using or monitoring the automation

What's not covered (post-window)

  • Changes caused by third-party platform updates (API changes, UI updates, etc.)
  • New features or expanded scope
  • Issues caused by changes made to the automation by the client

Ongoing maintenance and future updates can be arranged as a separate engagement.

Data & Security

I take the security of your business data seriously. Here is how I handle information during a project:

  • Only the minimum access needed to build the automation is requested
  • Credentials and API keys are stored securely and never shared
  • After project delivery, access can be revoked at your request
  • Client data is never used for any purpose other than the agreed project
  • Sensitive business information is treated as strictly confidential

Access Requests

To build automations that connect to your accounts, I'll typically need some form of access. This is standard for any automation development work.

How access is typically granted

  • OAuth Connections: Connecting tools via Zapier or Make.com uses secure OAuth ‚Äî no passwords are shared
  • API Keys: For direct API integrations, I'll ask for a read/write API key with the minimum required scope
  • Shared Access: For platforms like Google Workspace, I may ask to be added as an editor on specific files only
  • Test Accounts: Where possible, I use a test/sandbox environment to develop before going live

You're always in control: You grant access, you monitor it, and you can revoke it at any time. I'll never ask for more permissions than strictly necessary.
